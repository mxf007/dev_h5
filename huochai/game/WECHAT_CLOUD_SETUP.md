# 微信云开发配置指南

## 1. 开通微信云开发

1. 登录微信公众平台，进入小程序管理后台
2. 在「开发」-「开发设置」中找到「云开发」，开通云开发服务
3. 获取云开发环境 ID

## 2. 配置项目

### 2.1 修改环境 ID

在 [src/CloudConfig.ts](file:///g:/dev_h5/huochai/game/src/CloudConfig.ts) 文件中，将 `你的云开发环境ID` 替换为你的云开发环境 ID：

```typescript
public static CLOUDBASE_ENV: string = '你的云开发环境ID';  // 例如: 'prod-xxxxx'
```

### 2.2 创建云开发目录结构

在小程序开发者工具中，右键项目根目录，选择「新建 Cloud Functions Directory」，创建云函数目录。

## 3. 创建数据库集合

在微信云开发控制台中，创建以下集合：

- `game_data`: 存储玩家的游戏数据

## 4. 部署说明

1. 确保在微信开发者工具中勾选「将 JS 编译成 ES5」
2. 上传代码时，确保包含云开发相关文件
3. 在小程序管理后台提交审核时，注意勾选云开发相关权限

## 5. 老数据兼容机制

系统具有完善的老数据兼容机制：

- **首次访问云端**：如果云端没有数据，系统会将本地存储的游戏进度上传到云端
- **云端数据较新**：如果云端数据的时间戳比本地新，系统会将云端数据合并到本地
- **本地数据较高**：如果本地某些数值（如积分、关卡进度）比云端高，系统会保留较高的数值，防止进度丢失
- **无缝切换**：无论用户何时开始使用云存储，都能保证数据不丢失

## 6. 重要配置说明

**务必在发布前完成以下配置**：

1. 在 [CloudConfig.ts](file:///g:/dev_h5/huochai/game/src/CloudConfig.ts) 中替换为真实的环境ID
2. 确保在云开发控制台创建了 `game_data` 数据库集合
3. 设置正确的数据库访问权限

## 7. 测试

1. 在微信开发者工具中测试云开发功能
2. 确保数据能够正常上传和下载
3. 检查错误处理是否正常
4. 验证老数据兼容机制是否正常工作

## 注意事项

- 仅在微信环境中启用云开发功能
- 非微信环境会自动降级使用本地存储
- 数据上传有频率限制，请合理安排上传时机
- 遵循微信云开发的使用规范和限制
